# 光子 IP-SK5 随机调换工具

这是一个 Python + Tkinter 图形界面工具，用来批量随机调换光子 IP-SK5 的代理 IP。

输入格式一行一条：

```text
ip|端口|用户名|密码|到期时间
```

示例：

```text
175.6.50.122|5262|iuaf13s1|iuaf13s1|2026-07-13
117.21.36.82|7766|iuaf13s10|iuaf13s10|2026-07-13
```

输入多少行，程序就会逐条处理多少条。成功的结果会在“更换后的新 IP”里一行一条显示；某条失败时会标记失败并继续处理后面的 IP。
输入框左侧会显示代理序号，右侧会显示每条代理当前状态，空行不会占用序号。状态包括 `待处理`、`更换中`、`等待生效`、`测试中`、`测试合格`、`更换失败`、`测试失败` 等。
点击 `清空导入` 可以清空上方导入表单和状态，方便导入新数据。

## 连通性测试

界面里可以配置：

- `测试网址`：默认 `https://www.usnbweb.red`
- `最大延迟(ms)`：默认 `1000`
- `最大更换次数`：默认 `20`
- `等待测试(s)`：默认 `10`，每条代理调换成功后先独立倒计时这么多秒，再开始测试。
- `测速连接 SOCKS5 时走本机真实网络`：默认开启，测速时会把连接 SOCKS5 代理服务器这一跳绑定到本机 Wi-Fi/以太网网卡，尽量避开本机 VPN 的智能路由。
- `本机网卡IP`：默认 `自动`，程序会自动检测非 VPN 网卡；如果自动检测不准，可以手动填写本机真实网络的 IPv4。

点击 `保存配置` 会把这些配置保存到当前目录的 `ip_exchange_config.json`。下次打开程序会自动加载，不需要重复设置测试网址、最大延迟、最大更换次数、等待测试时间、本机真实网络开关和本机网卡 IP。

点击“调换+测试”时，程序会按流水线执行：

1. 调换接口串行执行，一次只调换一条代理。
2. 某条代理调换成功后，立刻在状态列开始独立倒计时，倒计时时长来自 `等待测试(s)`。
3. 该条倒计时结束后马上并发测试，不需要等待其他代理全部调换完成。
4. 合格就输出；不连通或延迟高于阈值的代理会进入调换队列排队，等轮到它时再次串行调换。
5. 每条代理都会重复“排队调换、独立等待、并发测试”，直到合格或达到最大更换次数。

如果你输入的是旧 IP，程序会先按用户名查询订单当前代理，再基于订单当前 IP 发起一次新的随机调换，保证输出结果一定是本轮新换出来的 IP。
如果达到最大更换次数后仍然不合格，结果列表会显示最后一次换出的新 IP，并在 `测试结果` 列标记失败原因；这样可以核查最终停在哪个代理上。

运行中可以点击“停止”按钮。程序会停止继续调换；如果某条正在倒计时或正在测速，会尽快中断。

这里的“ping”是通过代理访问目标网址的 HTTP 往返耗时，不是 ICMP ping；SOCKS5 代理本身不能直接做 ICMP ping，这种方式更接近真实访问网站的速度。

每次运行的日志和最终输出会写入当前目录的 `run_results.log`，方便事后核查每条 IP 是否换新、是否等待、是否测速、是否合格输出。

## 结果列表

“更换后的新 IP”会以列表显示，包含：

- `序号`
- `代理信息`：`ip|端口|用户名|密码|到期时间`
- `地区`：来自订单/节点接口返回的地区或节点信息
- `测试结果`：连通性、HTTP 状态、延迟等结果

结果列表的 `序号` 与上方输入框左侧的序号一一对应：上方第 1 条代理换出来的新 IP 会显示为下方第 1 条，上方第 2 条对应下方第 2 条。新的批量任务开始时会清空上一批结果，避免旧结果混在一起。

点击上方“复制结果”会复制全部代理信息，不会复制地区和测试结果。

在结果列表上右键，可以使用：

- `复制选中单条代理`
- `调换+测试`

右键菜单里的“调换+测试”会对当前选中的一条结果重新执行：换新、按 `等待测试(s)` 倒计时、测试连通性，直到合格或达到最大更换次数。无论最终合格还是不合格，都会在原位置显示最后一次换到的新 IP；不合格时会在 `测试结果` 列标记失败原因。

## 一键启动

双击：

```text
一键启动.vbs
```

这个脚本使用 `pythonw.exe` 或 `pyw.exe` 启动 `start_gui.pyw`，不会弹出黑色终端窗口。

如果双击后仍然没有界面，请查看当前目录里的：

```text
startup_error.log
```

里面会记录具体的 Python 启动错误。

## 首次安装依赖

第一次使用前需要安装依赖：

```powershell
pip install -r requirements.txt
python -m playwright install chromium
```

如果电脑已经安装 Edge 或 Chrome，通常第二条可以不执行；程序会优先尝试调用本机 Edge/Chrome。

## 登录

第一次使用先点“打开/登录网站”，在弹出的浏览器里登录网站。

登录态会保存在当前目录的 `browser_profile` 文件夹中，之后点击“随机调换”或“调换+测试”会直接在后台处理，不会打开网页窗口。

## 后台更换

程序固定使用后台接口模式，调用该网页前端使用的 `/node/orderDetail` 和 `/node/exchangeRandom` 接口。批量调换时不弹浏览器页面；如果登录态失效，会提示先点“首次登录/更新登录态”。

测速使用 Windows 自带的 `curl.exe`。程序会依次尝试多种 SOCKS5 连接方式，包括 `socks5h://用户名:密码@ip:端口`、`--socks5-hostname + --proxy-user`、`socks5://...`，避免某一种底层写法不兼容导致全部误判为不连通。

如果勾选了“测速连接 SOCKS5 时走本机真实网络”，程序会给 `curl.exe` 增加 `--interface 本机网卡IP`。这只影响本机连接 SOCKS5 代理服务器的出口；目标网站最终仍由 SOCKS5 代理出口访问。

请只用于你自己有权管理的账户和代理资源，并遵守目标网站规则。
